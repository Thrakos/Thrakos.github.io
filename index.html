Projects

Sidequests

A fun social media type app to keep track of all your fun activities!

This project was a simple simulation of a social media app that lets the user upload images and videos of fun "sidequests" after they are completed. The app keeps track of the user’s sidequests, and also provides an update on what the user’s friends are up to. It functions similarly to Instagram, but focuses more on the user’s sidequests to-do list.

In developing this app, I used a virtual "phone" simulator. Each view in the app is simulated by an individual window. The views included user’s profile, user’s feed and user’s to-do list. The “Profile” view provides an interface to enter/edit user’s name and bio, and view any sidequests the user has completed. The “Feed” view shows the sidequests the user’s friends have recently completed, and the “To-Do List” view keeps track of the tasks the user wants to complete.

This project was developed in Java, using the swing GUI toolkit. The development had to follow certain methodologies such as test-driven development. Design patterns such as the MVC pattern were also used to help with user input.

RISC-V Pipelined Processor

A five-stage processor for executing RISC-V instructions.

This project involved designing and implementing a pipelined RISC-V processor with support for forwarding and stalls. The processor spec included a five-stage pipelined architecture with instruction fetch, decode and execute, and memory access and write-back stages. This design had to ensure efficient instruction execution by employing overlapping stages to increase throughput. The processor also had to be capable of handling data and control hazards to avoid unnecessary stalling.

The project was developed using SystemVerilog. We started with a basic implementation that was progressively refined by adding essential components such as pipeline stages and a hazard detection unit. Testing and verification were conducted using Questa and a custom SystemVerilog testbench to ensure adherence to all functional requirements.

Working on this high-performance RISC-V processor project, I learned about complex digital design techniques such as pipelining as well as test and verification methodologies.

Reaction Testing Game

A fun game built using FreeRTOS

This project was a game designed to test and improve reaction time by flashing different patterns of colored lights. Each pattern corresponded to a specific number of button presses that the user needed to complete within a set time. The game included features such as score tracking, increasing difficulty levels, and randomized light sequences, making it both challenging and engaging for players.

The game was developed using the STM32CubeIDE for the STM32 Discovery Board, leveraging FreeRTOS to manage tasks and timing. The implementation involved creating a responsive system that handled real-time input, controlled LED patterns, and adjusted difficulty dynamically.

Through this project, I gained hands-on experience with embedded systems, task management in FreeRTOS, and real-time programming while also strengthening my skills in creating interactive applications on microcontroller platforms.
